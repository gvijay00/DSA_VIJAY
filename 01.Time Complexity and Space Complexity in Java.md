# Time Complexity and Space Complexity in Java


---

# 1. What is Complexity?

When we write a program, we mainly care about:

1. **How fast the program runs** → **Time Complexity**
2. **How much memory it uses** → **Space Complexity**

These help us measure the **efficiency** of an algorithm.

---

# 2. Why Complexity is Important?

Suppose you have:

* 10 numbers → any code works fast
* 10 million numbers → bad code becomes very slow

Example:

* Linear Search on 1 crore elements → very slow
* Binary Search on 1 crore elements → super fast

So companies like Google, Amazon, and Microsoft care heavily about complexity.

---

# 3. Time Complexity

## Definition

Time Complexity tells:

> How the execution time grows when input size increases.

Input size is represented by:

# `n`

---

# 4. Space Complexity

## Definition

Space Complexity tells:

> How much extra memory an algorithm uses as input size grows.

---

# 5. Big-O Notation

We use **Big-O notation** to represent complexity.

Examples:

| Complexity | Name         |
| ---------- | ------------ |
| O(1)       | Constant     |
| O(log n)   | Logarithmic  |
| O(n)       | Linear       |
| O(n log n) | Linearithmic |
| O(n²)      | Quadratic    |
| O(2ⁿ)      | Exponential  |
| O(n!)      | Factorial    |

---

# 6. Understanding Time Complexity Step by Step

---

# O(1) — Constant Time

Execution time does NOT depend on input size.

## Example

```java
int a = 10;
System.out.println(a);
```

Even if input is 10 or 1 crore → same time.

## Array Access

```java
int[] arr = {10,20,30};
System.out.println(arr[1]);
```

Accessing by index is always constant.

# Complexity

```text
Time  : O(1)
Space : O(1)
```

---

# O(n) — Linear Time

Time grows linearly with input size.

## Example

```java
for(int i=0;i<n;i++){
    System.out.println(i);
}
```

If:

* n = 10 → loop runs 10 times
* n = 1000 → loop runs 1000 times

# Complexity

```text
Time : O(n)
```

---

# O(n²) — Quadratic Time

Nested loops.

## Example

```java
for(int i=0;i<n;i++){
    for(int j=0;j<n;j++){
        System.out.println(i + " " + j);
    }
}
```

If n = 5:

```text
5 × 5 = 25
```

If n = 100:

```text
100 × 100 = 10000
```

# Complexity

```text
O(n²)
```

---

# O(log n) — Logarithmic Time

Very efficient.

Input keeps reducing by half.

## Best Example → Binary Search

```java
int binarySearch(int[] arr,int target){
    int low = 0;
    int high = arr.length-1;

    while(low <= high){
        int mid = (low + high)/2;

        if(arr[mid] == target)
            return mid;

        else if(arr[mid] < target)
            low = mid + 1;

        else
            high = mid - 1;
    }

    return -1;
}
```

---

## Why O(log n)?

Each step removes half data.

Example:

```text
1024
512
256
128
64
32
16
8
4
2
1
```

Only 10 steps!

---

# Binary Search Visualization

f(n)=\log_2 n

---

# O(n log n)

Seen in efficient sorting algorithms.

Examples:

* Merge Sort
* Quick Sort (average)
* Heap Sort

---

# Merge Sort Example

```java
void mergeSort(int[] arr){
    // divide
    // sort
    // merge
}
```

Complexity:

```text
O(n log n)
```

Very efficient for large data.

---

# O(2ⁿ) — Exponential Time

Very slow.

Usually seen in recursion without optimization.

## Fibonacci Recursive

```java
int fib(int n){
    if(n <= 1)
        return n;

    return fib(n-1) + fib(n-2);
}
```

Calls increase exponentially.

---

# Recursive Fibonacci Growth

T(n)=T(n-1)+T(n-2)

---

# O(n!) — Factorial Time

Worst complexity.

Seen in permutations.

## Example

```java
abc
acb
bac
bca
cab
cba
```

For 3 chars:

```text
3! = 6
```

For 10 chars:

```text
10! = 36,28,800
```

Extremely huge.

# 7. Rules for Calculating Time Complexity

---

# Rule 1: Ignore Constants

```java
for(int i=0;i<n;i++){}
for(int i=0;i<n;i++){}
```

Actual:

```text
O(2n)
```

Simplified:

```text
O(n)
```

---

# Rule 2: Different Variables

```java
for(int i=0;i<n;i++){}
for(int j=0;j<m;j++){}
```

Complexity:

```text
O(n + m)
```

---

# Rule 3: Nested Loops Multiply

```java
for(int i=0;i<n;i++){
    for(int j=0;j<n;j++){}
}
```

```text
O(n²)
```

---

# Rule 4: Sequential Loops Add

```java
for(int i=0;i<n;i++){}
for(int j=0;j<n;j++){}
```

```text
O(n+n)
= O(2n)
= O(n)
```

---

# 8. Space Complexity

---

# O(1) Space

No extra memory.

```java
int sum = 0;
```

---

# O(n) Space

Extra array/list.

```java
int[] temp = new int[n];
```

---

# Recursive Stack Space

Every recursive call uses stack memory.

## Example

```java
void fun(int n){
    if(n==0)
        return;

    fun(n-1);
}
```

Space:

```text
O(n)
```

Because function calls are stored in stack.

# 11. Amortized Complexity

Important in Java Collections.

Example:

```java
ArrayList<Integer> list = new ArrayList<>();
```

Normally:

```text
add() = O(1)
```

But sometimes resizing happens.

Then:

```text
O(n)
```

But average over many operations:

```text
Amortized O(1)
```

# 12. Recursive Complexity

---

# Example

```java
void fun(int n){
    if(n==0)
        return;

    fun(n-1);
}
```

## Time

```text
O(n)
```

## Space

```text
O(n)
```

Because recursion stack stores calls.

---

# Tree Recursion

```java
fib(n-1) + fib(n-2)
```

Creates recursion tree.

Complexity becomes exponential.

---

# 13. Master Theorem (Advanced)

Used for Divide & Conquer algorithms.

General form:

T(n)=aT\left(\frac{n}{b}\right)+f(n)

Examples:

* Merge Sort
* Quick Sort
* Binary Search

# 16. Tips to Reduce Complexity

---

## Use Hashing

Replace:

```text
O(n²)
```

with:

```text
O(n)
```

---

## Use Binary Search

Replace:

```text
O(n)
```

with:

```text
O(log n)
```

---

## Avoid Unnecessary Nested Loops

Nested loops often cause:

```text
O(n²)
```

---

## Use Sliding Window

Optimizes many array/string problems.

---

## Use Prefix Sum

Avoid repeated calculations.
